This document explains how to run tests for the database services.

Files:

Running Tests
=============

1. Install test dependencies:
   pip install pytest pytest-asyncio pytest-mock

2. Run all tests:
   pytest backend/ -v

3. Run specific test file:
   pytest backend/test_bot_service.py -v

4. Run specific test class:
   pytest backend/test_bot_service.py::TestBotServiceCreate -v

5. Run with coverage:
   pytest backend/ --cov=app/db --cov-report=html

6. Run tests matching pattern:
   pytest backend/ -k "test_create" -v

7. Run async tests only:
   pytest backend/ -m asyncio -v

        # ... test code ...

Test Markers
============

Mark tests with pytest markers:

    @pytest.mark.unit
    async def test_something():
        pass

    @pytest.mark.integration
    async def test_something():
        pass

    @pytest.mark.asyncio
    async def test_something():
        pass

Run tests with specific markers:

    pytest backend/ -m unit -v          # Only unit tests
    pytest backend/ -m integration -v   # Only integration tests
    pytest backend/ -m asyncio -v       # Only async tests

Debugging Tests
===============

Print debug info in tests:

    @pytest.mark.asyncio
    async def test_something(self, sample_bot):
        print(f"Bot: {sample_bot}")
        assert sample_bot is not None

Run with print output:

    pytest backend/ -v -s  # Show print statements

Run with detailed assertion info:

    pytest backend/ -vv  # More verbose

Use pdb debugger:

    import pdb
    
    @pytest.mark.asyncio
    async def test_something():
        pdb.set_trace()  # Stops execution here
        # Use 'c' to continue, 'n' for next line, etc.

CI/CD Integration
=================

Add to GitHub Actions (.github/workflows/tests.yml):

    name: Tests
    on: [push, pull_request]
    jobs:
      test:
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v2
          - uses: actions/setup-python@v2
            with:
              python-version: '3.9'
          - run: pip install -r requirements-test.txt
          - run: pytest backend/ --cov=app/db

Add to pre-commit hook (.git/hooks/pre-commit):

    #!/bin/bash
    pytest backend/ --quiet || exit 1

Best Practices
==============

1. Test one thing per test:
   ✓ test_create_bot_basic
   ✗ test_create_and_update_and_delete_bot

2. Use descriptive test names:
   ✓ test_update_bot_config_blocked_when_running
   ✗ test_update_config

3. Test both success and failure paths:
   ✓ test_delete_bot_success + test_delete_bot_fails_if_running
   ✗ only test_delete_bot

4. Use fixtures for reusable data:
   ✓ Use sample_bot_data fixture
   ✗ Create new data in every test

5. Mock external dependencies:
   ✓ Mock MongoDB collections
   ✗ Use real database in tests

6. Test ownership validation:
   ✓ Verify user_id filters in queries
   ✗ Skip ownership tests

7. Test error cases:
   ✓ Test invalid input, missing data, permission errors
   ✗ Only test happy path

Troubleshooting
===============

Issue: \"No module named 'pytest'\"
Solution: pip install pytest pytest-asyncio

Issue: \"RuntimeError: Event loop is closed\"
Solution: Ensure async tests use @pytest.mark.asyncio

Issue: \"AssertionError: assert None is not None\"
Solution: Check mock setup - verify mocks are configured before test runs

Issue: \"Tests pass locally but fail in CI\"
Solution: Check Python version, dependencies, environment variables

Issue: \"Test hangs indefinitely\"
Solution: Add timeout: pytest backend/ --timeout=60
