════════════════════════════════════════════════════════════════════════════════
🛠️ SCRIPTS CRIADOS
════════════════════════════════════════════════════════════════════════════════

1. STRESS TEST - Locust Framework
   Arquivo: backend/tests/stress/stress_test.py (115 linhas)
   Função: Simula 100 usuários fazendo login e consumindo API
   Endpoints Testados:
     - POST /auth/login (autenticação)
     - GET /me (perfil)
     - GET /bots (listagem de bots)
     - GET /analytics/dashboard (analytics)
     - GET /trading/history (histórico)
     - GET /health (health check)
   Validação: Security headers em cada resposta
   Uso:
     pip install locust
     locust -f backend/tests/stress/stress_test.py --host=http://localhost:8000 --users 100 --spawn-rate 10 --headless --run-time 5m --csv=stress_results

2. WEBSOCKET LOAD TEST - Async Test
   Arquivo: backend/tests/integration/test_websocket_load.py (340 linhas)
   Função: Abre 50 conexões WebSocket simultâneas
   Validação:
     - Latência de entrega < 200ms (P95)
     - Sem perda de mensagens
     - Conexões limpas
   Métricas:
     - P50, P95, P99 latência
     - Min, Max, Média
     - Taxa de erro
   Uso:
     pip install websockets
     python backend/tests/integration/test_websocket_load.py

3. RESOURCE MONITORING - System Monitor
   Arquivo: backend/app/core/monitoring.py (320 linhas)
   Função: Monitora recursos do sistema em tempo real
   Integração: Automática no backend/app/main.py
   Coleta A Cada 60 Segundos:
     - Memória RSS (MB)
     - CPU (%)
     - Threads (count)
     - File descriptors/Conexões (count)
   Output: Relatório detalhado ao fazer shutdown
   Análise:
     - Detecta vazamento de memória
     - Calcula percentis de CPU
     - Verifica picos de threads
     - Monitora conexões abertas

4. SECURITY HEADERS VALIDATION - Security Test
   Arquivo: backend/tests/security/test_headers_validation.py (310 linhas)
   Função: Valida 7 security headers em múltiplos endpoints
   Headers Validados:
     ✅ Strict-Transport-Security (HSTS)
     ✅ X-Content-Type-Options
     ✅ X-Frame-Options
     ✅ X-XSS-Protection
     ✅ Content-Security-Policy
     ✅ Referrer-Policy
     ✅ Permissions-Policy
   Endpoints Testados:
     - GET /health
     - GET /docs
     - GET /redoc
   Testes de Performance:
     - 10 requisições por endpoint
     - Média, mediana, min, max
     - Sem overhead perceptível
   Uso:
     pip install requests
     python backend/tests/security/test_headers_validation.py
