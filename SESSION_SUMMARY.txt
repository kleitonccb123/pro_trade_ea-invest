Key Integration Points
======================

The Complete Flow:

    HTTP Request (POST /api/orders/place)
           ↓
    AuthorizationMiddleware (validate Bearer token)
           ↓
    RequestLoggingMiddleware (LogSanitizer removes secrets)
           ↓
    /api/orders/place Router
           ↓
    OrderService.validate_and_execute_order()
           ↓
    RiskManager.validate_order() [FASE 1]
           ├─ Check account balance
           ├─ Check order size limits
           ├─ Check leverage limits
           └─ Return (is_valid, error_message)
           ↓
    OrderManager.execute_order() [FASE 1]
           ├─ Create KuCoinRawClient
           ├─ Call KuCoin API
           ├─ Retry with exponential backoff
           └─ Return OrderResult with exchange_order_id
           ↓
    Update MongoDB Orders Collection
           ├─ Set exchange_order_id
           ├─ Set status to OPEN
           ├─ Update updated_at
           └─ Create Position if not exists
           ↓
    HTTP Response with order_id, status, exchange_order_id


Error Flow:

    If RiskManager rejects:
    → Update order status to REJECTED
    → Return error to client
    → LogSanitizer removes sensitive details from logs

    If OrderManager fails:
    → Increment retry_count
    → Update order status to FAILED
    → Store error_message in database
    → Return error to client


Real-Time PnL Updates:

    Current price received (from WebSocket/API)
           ↓
    PositionService.update_position_price(current_price)
           ↓
    Calculate unrealized PnL:
    • For LONG: (current_price - entry_price) * size
    • For SHORT: (entry_price - current_price) * size
           ↓
    Update position in MongoDB atomically
           ↓
    WebSocket broadcasts position update to client


Per-User Data Isolation:

    Every query includes user_id filter:
    
    db.bots.find({"user_id": "user123"})
    db.orders.find({"user_id": "user123"})
    db.positions.find({"user_id": "user123"})
    db.trades.find({"user_id": "user123"})
    
    Prevents cross-user data leakage ✓
    Prevents unauthorized data access ✓

Deployment Guide
================

Development:
1. python -m uvicorn app.main:app --reload
2. API available at http://localhost:8000
3. Docs available at http://localhost:8000/docs

Production:
1. Set environment variables:
   - DATABASE_URL=mongodb+srv://...
   - CREDENTIAL_ENCRYPTION_KEY=<fernet_key>
   - GOOGLE_CLIENT_ID=...
   
2. Start with gunicorn:
   gunicorn -w 4 -k uvicorn.workers.UvicornWorker app.main:app

3. Configure reverse proxy (nginx/Apache)

4. Enable HTTPS/TLS

5. Setup monitoring & alerts

6. Configure database backups
